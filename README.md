# Maria Talks Tech

Static site with Tailwind CSS and a tiny TypeScript entry point. Local development uses Tailwind CLI and `live-server`.

## Quick start

```bash
cd /Users/mariablair/Desktop/maria-talks-tech
npm install
npm run dev
```

Then open `http://127.0.0.1:8080/`. The server watches `public/` and `src/`.

## Scripts

- `npm run dev`: runs Tailwind in watch mode and starts `live-server` on port 8080.
- `npm run build`: compiles `src/styles/input.css` → `public/assets/styles.css` (minified).
- `npm run format`: formats files with Prettier.

## Project layout

```
public/                # Static assets served in dev and deployed as-is
  assets/styles.css    # Generated by Tailwind CLI
  index.html           # Main HTML (uses Tailwind CDN for quick preview)
src/
  styles/input.css     # Tailwind source (imports @tailwind)
  scripts/main.ts      # Place TS/JS here for future interactivity
```

Note: For quick previews, `public/index.html` currently loads Tailwind via CDN. For production, build CSS and switch the `<link>` to `public/assets/styles.css` if you prefer fully local assets.

## Requirements

- Node.js 18+ and npm
- macOS/zsh environment (project rules assume this)

## Publishing to GitHub

The repo is configured to use SSH by default:

```
git remote -v
# origin  git@github.com:mashablair/maria-talks-tech.git
git push -u origin main
```

If you haven’t added an SSH key yet:
1) `ssh-keygen -t ed25519 -C "you@example.com"`
2) Add the public key (`~/.ssh/id_ed25519.pub`) at GitHub → Settings → SSH and GPG keys.
3) Test: `ssh -T git@github.com`

## Deploy options

- GitHub Pages: serve the `public/` folder.
- Cloudflare Pages/Netlify/Vercel: set the output directory to `public/`. Run `npm run build` in the build step to generate `assets/styles.css`.
